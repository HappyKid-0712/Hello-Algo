### 020

#### 题目描述

给定 $N$ 个朋友圈，每个朋友圈有 $k$ 个人。随后给定 $M$ 个需要查询的 ID。 要求找出这 $M$ 个人中**没有朋友**的人（即不属于任何人数大于 $1$ 的朋友圈的人），并按查询顺序输出他们的 ID。

> **特殊要求 1：** ID 为 5 位数字，输出时必须保留前导零。
>
> **特殊要求 2：** 如果同一个人被查询了多次，且他没有朋友，**只能输出一次**。
>
> **特殊要求 3：** ID 之间用空格分隔，行末不得有额外空格。如果没有这样的人，输出 `No one is handsome`。

#### 核心数据结构与思路

本质就是一个**集合的查找和去重**的问题。

- 使用哈希表`unordered_map`
- 算法流程：
  1. 先遍历朋友圈，把有朋友的人丢进hash打上`true`标记。
  2. 遍历查询列表，如果该人在哈希表上不存在，则存入` vector<string>res`，并立刻将该人打入map里面标记为有朋友（**只是为了防止重复查询**）。
  3. 最后输出`res`。

> 1. 只有 1 个人的朋友圈不算“有朋友”
> 2. 重复查询的去重
> 3. 行末无空格格式
